---
title: "Initiate Payment"
---

The **Standard Checkout** endpoint is used to create a payment session and generate a secure checkout link for your customers. This endpoint requires a valid Bearer token and merchant identification.

---

| **Environment** | **HTTP Method** | **API** |
| :-- | :-- | :-- |
| `Sandbox` | `POST` | https://api-sand.originpay.in/v2/checkout/pay |
| `Production` | `POST` | https://api.originpay.in/v2/checkout/pay |

## 1. Endpoint Details

| Attribute | Value |
| --- | --- |
| **URL** | `/v2/checkout/pay` |
| **Method** | `POST` |
| **Content-Type** | `application/json` |
| **Authentication** | Bearer Token (Authorization Header) |

---

## 2. Required Headers

| Header | Value | Description |
| --- | --- | --- |
| `Authorization` | `Bearer <access_token>` | Access token generated from `/v2/auth/token` |
| `X-MERCHANT-ID` | `<your_merchant_id>` | Unique merchant identifier |

---

## 3. Request Body

The request must be sent in JSON format.

### Core Fields

| Field | Type | Required | Description |
| --- | --- | --- | --- |
| `amount` | number | Yes | Amount in paise (1 INR = 100 paise). Range: 1 to 1,000,000 |
| `currency` | string | No | Currency code. Default: `"INR"` |
| `merchantOrderId` | string | Yes | Unique order ID from your system (alphanumeric) |
| `expireAfter` | number | No | Expiry time in seconds (300–900). Default: 300 |
| `customer` | object | Yes | Customer details |
| `paymentInfo` | object | Yes | Payment configuration |
| `paymentHandler` | object | Yes | Must be `{ "type": "STANDARD" }` |

---

### customer Object

| Field | Type | Required | Description |
| --- | --- | --- | --- |
| `name` | string | Yes | Full name of the customer |
| `email` | string | Yes | Valid email address |
| `phone` | string | Yes | Mobile number (max 15 characters) |

---

### paymentInfo Object

| Field | Type | Required | Description |
| --- | --- | --- | --- |
| `methods` | array | Yes | Allowed: `upi`, `card`, `netbanking`, `wallet` |
| `remark` | string | No | Transaction note or purpose |
| `redirectUrl` | string | Yes | URL where user is redirected after payment |

---

### Custom Fields

- You can pass up to **5 additional fields** at the root level
- Keys must be alphanumeric or underscores (max 25 characters)
- Values must be strings (max 25 characters)

---

## 4. Success Response

**Status Code:** `200 OK`

```json
{
  "success": true,
  "statusCode": 200,
  "msg": "Order created successfully",
  "data": {
    "merchantOrderId": "TXN_98765",
    "paymentUrl": "https://originpay.in/checkout/standard/pay_v2?token=eyJhbGci..."
  }
}
```

---

## 5. Error Codes

| Status Code | Message (`msg`) | Description |
| --- | --- | --- |
| `400` | Amount must be between 1 and 1000000 paise | Invalid amount |
| `400` | Valid customer email is required | Invalid email format |
| `400` | Payment handler is required and must be STANDARD | Invalid `paymentHandler` |
| `409` | Merchant order id already exists | Duplicate order ID |
| `409` | Transaction limit reached for today | Daily limit exceeded |
| `503` | Payment APIs are currently down for maintenance | System under maintenance |

---

## 6. Implementation Example

### JavaScript (Fetch)

```javascript
const requestBody = {
  amount: 50000,
  currency: "INR",
  merchantOrderId: "ORDER12345",
  expireAfter: 600,
  customer: {
    name: "John Doe",
    email: "john@example.com",
    phone: "9876543210"
  },
  paymentInfo: {
    methods: ["upi", "card", "netbanking", "wallet"],
    remark: "Subscription Payment",
    redirectUrl: "https://yourwebsite.com/callback"
  },
  paymentHandler: {
    type: "STANDARD"
  },
  udf1: "Additional Info 1",
  udf2: "Additional Info 2",
  udf3: "Additional Info 3",
  udf4: "Additional Info 4",
  udf5: "Additional Info 5"
};

const response = await fetch('https://api.originpay.in/v2/checkout/pay', {
  method: 'POST',
  headers: {
    'Content-Type': 'application/json',
    'Authorization': 'Bearer YOUR_ACCESS_TOKEN',
    'X-MERCHANT-ID': 'YOUR_MID'
  },
  body: JSON.stringify(requestBody)
});
```

---

## Important Notes

- Ensure `merchantOrderId` is unique for every request
- The generated `paymentUrl` is single-use and expires based on `expireAfter`
- Always verify payment status via backend or webhook
- Do not rely solely on frontend redirection for confirmation

---

This endpoint is responsible for initiating the payment flow and securely redirecting users to the hosted checkout page.
