## Integration Steps

Follow these steps to integrate **Originpay API v2** into your application using the Standard Checkout flow.

---

### 1. Generate Access Token (Authentication)

Before making any payment request, you must generate a **Bearer token** using your API credentials.

* **Endpoint:** `POST /v2/auth/token`
* **Content-Type:** `application/x-www-form-urlencoded`

#### Required Parameters:

* `public_key`
* `secret_key`

#### What Happens:

* Your credentials are validated
* Merchant status, KYC, and gateway availability are checked
* A **JWT Bearer token** is issued (valid for 3600 seconds)

#### Usage:

Include the token in all subsequent requests:

```
Authorization: Bearer <your_token>
```

---

### 2. Create Payment Session

Once authenticated, create a payment session using the checkout API.

* **Endpoint:** `POST /v2/checkout/pay`
* **Content-Type:** `application/json`

#### Required Headers:

* `Authorization: Bearer <your_token>`
* `X-MERCHANT-ID: <your_merchant_id>`

#### Request Includes:

* Order amount
* Customer details (name, email, phone)
* Redirect URL
* Optional metadata (custom fields)

#### What Happens:

* A transaction is created in the system
* A secure **checkout session** is generated
* A `paymentUrl` is returned

---

### 3. Redirect Customer to Checkout

Use the `paymentUrl` received in the response to redirect your user to the hosted Originpay checkout page.

#### Example:

* Web: Redirect using browser navigation
* Mobile App: Open in WebView or external browser

#### What Happens:

* Customer selects payment method (UPI, Card, etc.)
* Payment is processed securely by Originpay

---

### 4. Handle Payment Completion

After payment, the user is redirected back to your provided `redirectUrl`.

#### Important:

* Do not rely only on frontend success response
* Always verify payment status from backend or webhook

---

### 5. Verify Payment Status (Recommended)

To ensure payment authenticity, verify the transaction:

* Use backend APIs or webhook data
* Match transaction ID and payment status

This prevents issues like:

* Fake success responses
* Tampered client-side data

---

### 6. Setup Webhooks (Recommended)

Webhooks allow your server to receive real-time payment updates.

#### Common Events:

* `payment.success`
* `payment.failed`
* `payment.pending`

#### Best Practices:

* Verify webhook signature
* Handle duplicate events (idempotency)
* Respond with HTTP `200` to acknowledge receipt

---

### 7. Handle Errors & Retries

Ensure your integration handles failures gracefully:

* Retry failed API calls when safe
* Log all request/response data
* Show proper error messages to users

---

## Integration Flow Summary

1. Generate Bearer Token
2. Create Payment Session
3. Redirect User to Checkout
4. Customer Completes Payment
5. Verify Payment via Webhook/API

---

By following these steps, you can securely integrate and start accepting payments using **Originpay API v2.
