---
title: "Order Status"
---

The **Check Order Status** endpoint allows you to retrieve the real-time status and details of a transaction using your `merchantOrderId`. This is commonly used for reconciliation, backend verification, and displaying payment results to users.

---

| **Environment** | **HTTP Method** | **API** |
| :-- | :-- | :-- |
| `Sandbox` | `POST` | https://api-sand.originpay.in/v2/order/status?order\_id={merchantOrderId} |
| `Production` | `POST` | https://api.originpay.in/v2/order/status?order\_id={merchantOrderId} |

## 1. Endpoint Details

| Attribute | Value |
| --- | --- |
| **URL** | `/v2/order/status?order_id={merchantOrderId}` |
| **Method** | `GET` |
| **Authentication** | Bearer Token (Authorization Header) |

---

## 2. Required Headers

| Header | Value | Description |
| --- | --- | --- |
| `Authorization` | `Bearer <access_token>` | Valid access token generated from Auth API |
| `X-MERCHANT-ID` | `<your_merchant_id>` | Unique merchant identifier |

---

## 3. Query Parameters

| Parameter | Type | Required | Description |
| --- | --- | --- | --- |
| `order_id` | string | Yes | The `merchantOrderId` used during payment creation |

---

## 4. Transaction Status Values

The API returns standardized transaction states using the `txnStatus` field:

| Status | Description |
| --- | --- |
| `SUCCESS` | Payment completed successfully |
| `FAIL` | Payment failed at bank or gateway |
| `REJECT` | Transaction rejected by system or risk checks |
| `CREATED` | Order created but payment not completed |
| `PENDING` | Transaction not found or still processing |

---

---

## 6. Error Responses

| Status Code | Message (`msg`) | Description |
| --- | --- | --- |
| `400` | order\_id query parameter is required | Missing `order_id` in request |
| `400` | X-MERCHANT-ID header is required | Missing merchant identifier |
| `401` | User key not found | Invalid or expired token |
| `403` | Business status is disabled | Merchant account restricted |
| `503` | Payment APIs are currently down for maintenance | System under maintenance |

---

## 7. Implementation Example

### cURL

```bash
curl -X GET "https://api.originpay.in/v2/order/status?order_id=ORDER12345" \
  -H "Authorization: Bearer YOUR_ACCESS_TOKEN" \
  -H "X-MERCHANT-ID: YOUR_MID"
```
