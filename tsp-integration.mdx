Welcome to the Originpay TSP API Documentation, your gateway to building secure, scalable, and seamless payment integrations for modern businesses.

Originpay TSP (Technology Service Provider) APIs enable platforms, fintechs, aggregators, and enterprise partners to programmatically manage merchants, initiate payments, verify transactions, and automate financial workflows through standardized REST APIs.

Whether you're onboarding merchants at scale, building custom checkout experiences, or integrating payment operations directly into your platform, Originpay provides the infrastructure required to simplify and automate the entire payment lifecycle.

This documentation covers authentication, merchant onboarding, payments, transaction management, callbacks, security best practices, and integration workflows.

## Base URL

| **Environment** | **HTTP Method** | **API** |
| :-- | :-- | :-- |
| `Sandbox` | `POST` | [https://api-sand.originpay.in](https://api-sand.originpay.in) |
| `Production` | `POST` | [https://api.originpay.in](https://api.originpay.in) |

All requests must be made over HTTPS.

## What is Originpay TSP?

Originpay TSP is a developer-first payment infrastructure platform designed to provide direct programmatic access to payment services and merchant operations.

Using Originpay APIs, partners can:

- Create and manage merchant accounts
- Initiate and process payment requests
- Handle transaction callbacks in real time
- Query transaction and settlement information
- Automate reconciliation and reporting
- Integrate payment workflows directly into applications

The APIs are built for reliability, scalability, and enterprise-grade security, making them suitable for payment facilitators, SaaS platforms, marketplaces, fintech applications, and large-scale businesses.
